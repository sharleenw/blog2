A bit of background

In early 2019, I created a blogdown blog with the help of Alison Hill‚Äôs great article Up & Running with blogdown that had 5 posts on it.

Recently, when I went to make a new blog post, I got the sense that the blog was going to break if I updated some packages.

I decided I could either try to fix the blog, or make a new one (with a new theme!) and copy over my old blog posts. I chose the second option.

This ended up being a lot of work (or, put another way, a great learning opportunity üòÇ), so I wanted to jot down some things I‚Äôve learned, in the hopes it might help someone else, or just myself, if I ever try to do anything blogdown-related again!

So, in no particular order, here are some things I‚Äôve learned!

Read the freakin‚Äô manual

For whatever reason, I forget about search engines and documentation when staring at the complex file structure of a new blogdown blog project folder.

But, as a note to my future self, read the docs! Everything got a lot easier once I remembered that üòÖ.

  • Hugo Tranquilpeak has a great user documentation section.

  • blogdown itself has a book that has everything you could ever need to know about blogdown.

  • Alison Hill has a fantastic series of posts on her blog called ‚ÄúA Spoonful of Hugo‚Äù (with at least these four articles):

    • Page Bundles
    • The netlify.toml File
    • Archetypes
    • Troubleshooting Your Build

And, just generally, including search terms like ‚Äúblogdown rmd‚Äù, ‚Äúhugo‚Äù, or ‚Äúhugo tranquilpeak‚Äù along with whatever my question is will get different types of answers, some that will be more geared towards R and some that will be geared more towards the website itself. I have found both types of searches to be helpful.

How can I get emojis to work‚ùó‚ùì

Are your emojis looking like :cry: when you want them to look like üòÑ?

With my last blog (Academic theme), emojis just worked. If I typed ‚Äú:smile:‚Äù into the text of my .Rmd file, the blog knew to convert it to üòÑ.

With my new theme (Tranquilpeak), the emojis of my posts were no longer working!

I found my solution at the bottom of the page here: Download Hadley‚Äôs emo package and type `r emo::ji("smile")` into the text of the .Rmd file.

It takes more keystrokes to write, but this way, I think the emojis will always work no matter which theme I‚Äôm using.

Use devtools::session_info() at the bottom of every post

When transferring over my posts, two of them no longer worked due to external package updates!

So, following the great example set by David Neuzerling‚Äôs blog (and others), I will be adding devtools::session_info() to the bottom of my posts. I mean, it won‚Äôt fix the fact that my code no longer works, but it adds an extra layer of proof that it did, in fact, work, at that particular time, given that particular set of described tools!

Making a template

Speaking of Alison Hill‚Äôs post on Archetypes (which references this blog post by Leo Collado-Torres), I‚Äôve added an archetype to my blog. An ‚Äúarchetype‚Äù being another word for template.

I created an archetype folder in my top-level directory, and added my template (called template.md) to that folder. This template archetype now pops up as an option when I use the RStudio Addin ‚ÄúNew Post‚Äù.

Here is the code that I used for my /archetype/template.md (modified from Leo Collado-Torres‚Äô archetype):

---
title: "{{ replace .TranslationBaseName "-" " " | title }}"
date: "{{ .Date }}"
author: Sharleen Weatherley
summary: "This is the summary that will appear on the main page."
draft: false

tags:
- tag1
- tag2

coverImage: http://res.publicdomainfiles.com/pdf_view/145/13978843819386.jpg
coverCaption: "A short caption (Source: The source)"
thumbnailImage: http://res.publicdomainfiles.com/pdf_view/145/13978843819386.jpg
thumbnailImagePosition: left
autoThumbnailImage: yes
coverSize: partial

blogdown::html_page:
  toc: no
---

What to touch and what not to touch

  • The actual blog content goes in the /content folder, with posts going in the /content/posts/ folder and any other type of content going in its own section (like /content/about/ for the About section.)
  • Use page bundles to keep the content of the /content folder organized, and follow Yihui‚Äôs advice to change the format of the permalinks to "/:year/:month/:day/:slug/" (Page Bundles).
  • The /public folder can be deleted before rebuilding your website. In fact, why not regularly restart R, delete the /public folder and then blogdown:::serve_site() to make everything start fresh! (Troubleshooting Your Build).
  • The /archetype folder needs to be top-level (I had to create it myself), as does the _output.yml file.
  • If you have images or CSS you want to include, create the appropriate folders in the /static folder (/static/images or /static/css), as these will be copied into the /public folder once the site is built (blogdown book).
  • Don‚Äôt touch the /theme folder! You should be able to do everything you need to do in the other folders.

Adding an ‚ÄúAbout‚Äù page

The ‚ÄúAbout‚Äù page is not really a post, it‚Äôs more of a standalone document. With the Tranquilpeak theme, there is actually an archetype called a ‚Äúpage‚Äù that you can use when creating it. The file structure I used to create it was /content/about/index.md.

There is a difference between .Rmd and .Rmarkdown (who knew?)

Tranquilpeak theme has a pretty syntax highlighting associated with it, but I couldn‚Äôt always get it to work and was mystified. Until I realized what was happening (by reading the blogdown book):

  • Files saved as .Rmarkdown get converted to .markdown first, before .html.
  • However, files saved as .Rmd go straight to .html.

The fancy syntax highlighting only works in .markdown files (which only happens if I save the files as .Rmarkdown and not .Rmd). Some of the other CSS only works for .markdown files too. (See the section on adding CSS.)

The blogdown book prefers .Rmd (as it allows for citations, which I have), so that‚Äôs what I‚Äôm using. I figured out the syntax highlighting in the next section.

Syntax highlighting

Thanks to the blogdown book, I found out I could create a _output.yml file in the root directory, and include a .Rmd-specific syntax highlighting template for all my files. My _output.yml file looks like:

blogdown::html_page:
  highlight: tango

Garrick Aden-Buie‚Äôs blog has a nice gallery comparingthe different syntax-highlighting options.

Relative links

On my original blog, I included all absolute links. Yikes! So now, thanks to Alison Hill‚Äôs Page Bundles post (that references the blogdown book), I now know how to make relative links. Basically, if I want to include relative posts, I just look at what the structure of the /public folder is, and use that as my relative link. For example, if I wanted to refer to this post, I would use /2020/07/08/how-to-remake-a-blogdown-blog-from-scratch/ as the link.

My settings for the ‚ÄúNew Post‚Äù Addin

  • Add a title. Author and Date should be filled in automatically
  • Subdirectory is ‚Äúpost‚Äù
  • I don‚Äôt include categories on my blog
  • Add some relevant tags
  • Use the template.md archetype
  • Keep the default slug
  • Delete the ‚Äúen-us‚Äù under Language
  • Format is .Rmd, not Rmarkdown
